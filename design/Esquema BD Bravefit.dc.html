ERP BRAVEFIT · DOCUMENTO TÉCNICO

Esquema de base de datos

PostgreSQL recomendado · v1 · julio 2026 · insumo para desarrollo en Claude Code
Principios: nada se repite (3ª forma normal) · catálogos parametrizables (etapas, estados, categorías y roles son tablas, no valores fijos — así agregar módulos o estados no requiere cambiar código) · llaves foráneas con índices para consultas ágiles · borrado lógico (activo/eliminado_en) para no perder historial.

0 · Núcleo: usuarios, roles y permisos

Preparado para los perfiles que definirás luego: cada rol tiene permisos por módulo (ver / crear / editar / aprobar) y campos ocultos.
usuarios
id PK · uuid
empleado_id FK→empleados (opcional)
rol_id FK→roles
email · hash_clave · nombre
activo · creado_en · ultimo_acceso
roles  +  permisos
roles: id PK · nombre (Admin, Ventas…)
permisos: id PK · rol_id FK→roles
  modulo (ej. 'ops', 'nomina')
  puede_ver · puede_crear · puede_editar · puede_aprobar
  campos_ocultos jsonb (ej. ["salario"])

5 · Integraciones y auditoría

integracion_eventos (id PK · sistema [shopify / siigo / whatsapp] · tipo_evento · payload jsonb · estado · procesado_en) — cola de sincronización: webhooks de Shopify entran aquí y un worker los convierte en pedidos_web/O.P.; las facturas se empujan a Siigo por su API; WhatsApp vía Cloud API oficial de Meta (sin riesgo de baneo).
pyg_mensual (id PK · periodo · concepto_id FK→conceptos_pyg · valor · cargado_por FK→usuarios · en) — el PyG del Dashboard/Finanzas NO se calcula: se carga manualmente desde el cuadro resumen que hace gerencia con datos de contabilidad (pantalla de captura o importación de Excel). conceptos_pyg parametriza filas, % meta y dirección (más/menos es mejor).
auditoria (usuario_id FK · tabla · registro_id · accion · cambios jsonb · en) — quién cambió qué, en todos los módulos.

6 · Flujo principal (relaciones clave)

oportunidad (CRM) → cotización → aprobada → orden_pedido → etapas → entrega → factura a Siigo
Shopify webhook → pedido_web → (pagado) → orden_pedido automática + descuento de inventario
consumo bajo buffer → alerta → solicitud_compra → comprado → recepción (faltantes → seguimiento) → entrada a inventario
O.P. entregada → encuesta post-entrega · reclamo → garantía ligada a la O.P. original
